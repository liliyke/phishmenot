This is an online checklist that needs to be passed to confirm an email is safe or give recommendations if any of the checklist fails

  • Are you able to view the email header?

    • Is there a RETURN-PATH header that is not matching the FROM header?
    • Are there any domains that do not match in the RECEIVED headers?
    • Is there a mismatch between the REPLY-TO and FROM headers?
    • Is there a RECEIVED-SPF header that cannot be verified using whois information?
    • Is there a DKIM header with the value "none‚Äù?
    • Are any X-HEADERS present to identify it as spam?

  • Are there any URLs or hypertext links in the email body of the message?

    • Is there an unfamiliar domain name when you hover over the links?
    • Copy the links and paste them into a separate environment, such as an online browser service like Browserling https://www.browserling.com/. Is it displaying any malicious behaviour, such as downloading files, navigating to the wrong domain, prompting for software installation, and so on?

  • Does the email have attachments?

    • Do you have an isolated environment where you can download it?
    • Can you contact your IT security or support team about it?
